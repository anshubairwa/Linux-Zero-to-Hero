# Linux-Zero-to-Hero

🐧 30-Day Linux Administration Challenge: Learn, Practice, and Master Linux Skills

Linux is the backbone of modern IT infrastructure—powering servers, cloud platforms, DevOps pipelines, and even embedded systems. If you are an aspiring system administrator, DevOps engineer, or IT professional, mastering Linux is non-negotiable.

But here’s the problem: where do you start, and how do you stay consistent?

That’s why I’ve designed a 30-Day Linux Administration Challenge—a structured, daily learning plan where you practice practical tasks, learn new commands, and build confidence as a Linux admin.

🚀 How the 30-Day Challenge Works

Time Needed: 30–45 minutes daily

Tools Required: A Linux system (Ubuntu/CentOS) or a virtual machine

Format: Each day covers a key Linux admin concept with commands and tasks

Goal: By the end of 30 days, you’ll have a strong foundation in Linux server administration

📅 The 30-Day Linux Administration Challenge
Week 1: Linux Basics & Navigation

Day 1: Introduction to Linux, history, and distributions

Day 2: File system hierarchy explained (/etc, /var, /home, /root)

Day 3: File and directory commands (ls, cd, pwd, cp, mv, rm)

Day 4: Working with text files (cat, less, head, tail)

Day 5: File permissions & ownership (chmod, chown, umask)

Day 6: User management (useradd, passwd, groups)

Day 7: Practice challenge: Create a new user with limited permissions

Week 2: Process & System Management

Day 8: Understanding processes (ps, top, htop, kill)

Day 9: Scheduling jobs with cron and at

Day 10: Monitoring logs (/var/log, journalctl, dmesg)

Day 11: Package management (APT/YUM/DNF basics)

Day 12: System services with systemctl

Day 13: Uptime, load average, and performance monitoring

Day 14: Practice challenge: Schedule a daily backup using cron

Week 3: Networking & Security

Day 15: Network basics (ifconfig, ip, ping, traceroute)

Day 16: Checking open ports (ss, netstat, nmap)

Day 17: SSH basics & key-based authentication

Day 18: Firewall management (ufw, iptables)

Day 19: Sudo privileges & securing root access

Day 20: Basic shell scripting for admins

Day 21: Practice challenge: Write a script to monitor disk usage

Week 4: Advanced Administration & Projects

Day 22: Disk management (df, du, mount, lsblk, fdisk)

Day 23: File compression & archiving (tar, gzip, zip)

Day 24: Backup strategies & automation

Day 25: System resource limits (ulimit, /etc/security/limits.conf)

Day 26: Configuring services (Nginx/Apache basics)

Day 27: Basic troubleshooting steps

Day 28: Security updates & patch management

Day 29: Real-world project: Set up a web server with a simple site

Day 30: Final challenge: Create a full backup & restore test

🎯 What You’ll Achieve After 30 Days

By completing this challenge, you will:
✅ Navigate and manage Linux systems confidently
✅ Understand system processes, services, and logs
✅ Secure and monitor your server
✅ Automate repetitive admin tasks with Bash scripts
✅ Deploy a basic server environment for real projects

📌 Final Thoughts

Linux administration might feel overwhelming at first, but with daily structured practice, you’ll gain the knowledge and muscle memory needed to become a confident sysadmin.
